Parse `$ARGUMENTS`. Route to the matching subcommand below.

## Subcommands

### `index <path>`
See [index.md](index.md) for full instructions.
Run `huh index --json <path>`, generate a semantic summary, save it with `huh save-index`.

### `changelog <trigger> [note]`
See [changelog.md](changelog.md) for full instructions.
Triggers: `milestone`, `finally-works`, `major-change`, `pre-compact`, `session-end`, `checkpoint`.
Record what changed, why it matters, and the outcome.

### `search <query>`
Run `huh search "<query>" --limit 20` and summarize the top results — title, layer, importance, first 2 sentences of content.

### `search-path <path>`
See [search.md](search.md). Check if a file or directory is already indexed before reading it.

### `tree [path]`
See [tree.md](tree.md). Show project structure with index status markers.

### `stats`
Run `huh stats` and display layer counts, cache size, file states.

### `reflect`
Run `huh reflect` and report how many L1/L2/L3 episodes were created.
Note: runs automatically on SessionEnd and PreCompact. Use mid-session if context is getting large.

### `prune`
Run `huh prune` and report episodes removed per layer.
Note: runs automatically on SessionEnd and PreCompact. Use mid-session only.

### `save-index --path <path> --level <level> --summary "<text>"`
Persist a semantic summary generated by Claude at the given level (symbol, file, dir, module, feature, project).
Run `huh save-index --path <path> --level <level> --summary "<text>"`. Confirm the episode ID saved.

### `save <text>`
Run `huh save "<text>" --importance 0.9 --permanent`. Confirm the episode ID saved.

## Episodic layer reference

When deciding what layer to write to or what decay to expect, see:
- [l0.md](l0.md) — raw episodes, 1-day decay, automatic
- [l1.md](l1.md) — session summaries, 1-week decay, promoted by reflect
- [l2.md](l2.md) — topic clusters, 1-month decay, promoted by reflect
- [l3.md](l3.md) — permanent, no decay, corrections and hard facts
