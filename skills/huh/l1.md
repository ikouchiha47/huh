# L1 — Session Summaries (180-day decay)

Consolidations of what happened across a session or a coherent unit of work.

## What belongs here

- End-of-session summary: what was worked on, what changed, what's pending
- A directory or module index entry (dir/module level from `/huh index`)
- A topic summary: "everything we learned about BLE power management this week"
- A resolved problem: what the problem was, what approaches failed, what worked
- A checkpoint created by `huh checkpoint`

## Decay

**1 week.** Survives across a sprint or short project phase. Feeds L2 clustering. Anything still relevant after a week should be promoted to L2.

## When to write

Write an L1 episode when a coherent unit of work is done:
- Session is ending or context is near 50%
- A subsystem is now understood well enough to summarize
- Multiple L0 episodes on the same topic exist and can be collapsed

`huh reflect` does this automatically — runs consolidation across recent L0 episodes.

Manual:
```bash
huh save "<session or topic summary>" --layer 1 --importance 0.7
```

Or for code:
```bash
huh save-index --path <dir_or_module_path> --level dir --summary "<summary>"
huh save-index --path <dir_or_module_path> --level module --summary "<summary>"
```

## What does NOT belong here

- A single specific event (→ L0)
- A long-running pattern or architectural insight (→ L2)
- Something that must never be forgotten (→ L3)
