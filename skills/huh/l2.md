# L2 — Topic Clusters (2-year decay)

Patterns, architectural insights, and lessons that span multiple sessions.

## What belongs here

- Architectural decisions: why the codebase is structured a certain way
- Recurring failure patterns: "X always breaks when Y changes"
- Feature-level or project-level code index entries
- Long-running design constraints: hardware limits, protocol quirks, performance ceilings
- Lessons learned from major debugging sessions or rewrites

## Decay

**1 month.** Covers the lifespan of a feature or milestone. Patterns that outlive a month belong in L3.

## When to write

Write an L2 episode when a pattern has been observed across multiple sessions or a major architectural truth has been established:
- A rewrite just completed and the lessons should outlive the session
- A `finally-works` changelog was recorded — extract the durable lesson into L2
- A design decision was made that future contributors need to understand

Manual:
```bash
huh save "<architectural insight or pattern>" --layer 2 --importance 0.8
```

For code:
```bash
huh save-index --path <feature_or_project_path> --level feature --summary "<summary>"
huh save-index --path <project_root> --level project --summary "<summary>"
```

`huh reflect` promotes dense L1 clusters to L2 automatically over time.

## What does NOT belong here

- A single session's observations (→ L0 or L1)
- Facts that must survive even if this project is archived (→ L3)
