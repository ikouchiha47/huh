# L3 — Permanent (no decay)

Facts, corrections, and constraints that must survive indefinitely.

## What belongs here

- Explicit user corrections: "don't do X, we got burned by that"
- Hard constraints that never change: hardware limits, protocol constants, legal/compliance requirements
- Validated facts: "this approach was confirmed to work in production"
- User preferences stated definitively: "always use Y pattern for Z"

## Decay

None. L3 episodes are never pruned by the decay job.

## When to write

Write L3 only when the user explicitly states something should be remembered permanently, or when a correction is made that must never be repeated:
- User says "remember this" / "never do X again"
- A hard-won lesson from a production incident
- A constraint that applies to every future session in this project

```bash
huh save "<fact or correction>" --layer 3 --importance 1.0 --permanent
```

The `--permanent` flag sets the `permanent` tag and bypasses the decay scorer on every future prune run.

## What does NOT belong here

- Observations that might become outdated (→ L0/L1)
- Architectural patterns that could change (→ L2)
- Anything speculative or unvalidated — L3 is ground truth, not hypothesis
