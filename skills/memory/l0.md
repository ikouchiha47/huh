# L0 — Raw Episodes (30-day decay)

Immediate observations: what just happened, what was tried, what the outcome was.

## What belongs here

- A specific bug that was found and fixed
- A failed approach and why it failed
- A tool call result that was surprising or important
- A user correction or preference stated explicitly
- A code symbol indexed for the first time (code_index category)
- A file read or written during this session

## Decay

**1 day.** Raw observations are ephemeral — they either get promoted to L1 by `huh reflect` or they expire.
Corrections and explicit user preferences are tagged `permanent` and do not decay.

`huh reflect` and `huh prune` run **automatically** on every `SessionEnd` and `PreCompact` hook — you do not need to call them manually. The `/memory reflect` and `/memory prune` commands exist only for mid-session use.

## Promotion to L1

`huh reflect` groups L0 episodes by session. When a session has **20 or more unsummarized L0 episodes** (no `parent_id` set), they are batched into an L1 summary. The L1 importance is the average importance of the batch, capped at 1.0. Each promoted L0 episode gets its `parent_id` set to the L1 episode — it won't be batched again.

Episodes that decay below `0.05` score (after ~3 days at 1-day half-life with no access) are archived and eventually deleted by `huh prune`.

## When to write

Write an L0 episode when something specific happened that a future session might need to know:
- "Tried X, it failed because Y"
- "User prefers Z"
- "This function does W — counterintuitive because..."

```bash
huh save "<content>" --layer 0 --importance <0.0-1.0>
```

Importance guide: 0.5 = routine observation, 0.7 = worth remembering, 0.9 = correction or key fact, 1.0 = permanent.

## What does NOT belong here

- Summaries of multiple events (→ L1)
- Patterns observed across sessions (→ L2)
- Facts that must never be forgotten (→ L3)
